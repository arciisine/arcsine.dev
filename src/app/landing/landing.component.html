ArcSine is about integrity, passion, excellence and bettering the world. I'm fully of the belief that the entire point of technology is to impact humanity in a meaningful way. We do not write software just to program, but to improve the lives of those around us.

My goal is to create software that not only pushes the bounds of what software is capable of, but also to meet real needs. As someone who has been writing software for over 20 years, there are so many systemic issues with how we develop software. Starting from the perspective that people are the end goal transforms how we visualize solutions.

Projects

Throughout my career, I've built many different tools, solutions, products with the express goal of making my job easier. More and more I've pushed to share these tools with other people, to enrich other's lives. I've gotten to explore how to take the tool from something that helps just me, to being something that helps others. Below are a sample of the open source projects I've created over the past few years.

Nodesh Source

A common problem I've faced in my career, is being able to share and maintain bash scripts. Nodesh is a library with the purpose of providing a shell-like programming experience within node. The library is written in typescript, and so the typing support is fantastic. It is async-generator oriented, meaning it streams iteratively, similar to how shell commands and piping work. The goal here is simplicity of usage as well as authoring.

VSCode npx-scripts Source

This is a sister project of nodesh, with the intent of facilitating the authoring process for npx-based scripts. The goal is to create a tool-chain that takes the pain out of script authoring, and sharing. This project combines two interests of mine, shell scripting and type-safety. This tool allows for acquisition of types for any library that supports global types.

Travetto Framework Source

I've worked on many different web-based solutions, using frameworks such as Spring, Struts, Django, Wordpress, Express, Flask, etc. One of the main take away I had from using these frameworks is the general difficulty of navigating features, as well as being able to iterate quickly. I took a vision of a responsive, simple, useful application framework and brought it to fruition.

VSCode Chronicler Source

In the process of generating documentation, I realized that recording screen activity is challenging. Especially when dealing with a cross platform development experience. There are many platform specific tools, but this falls flat when trying to work in a heterogeneous development environment. So, in the process I decided to build my own tool, to facilitate recording, ensuring it worked on all platforms.

Material Theme Generator Source

During the course of my professional career, I've gotten to use Angular, and one of the benefits is the Material integration. This is essentially a UI Toolkit based on the material design spec. The primary drawback is the difficulty in configuring the theme of the toolkit. Working through use cases and feedback from designers, I created a tool that allows for real-time theme configuration with the ability to share, and export themes.
